## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Wikidata SPARQL for relations | High precision, zero cost, deterministic, reproducible for thesis | LLM-based RE (noisy), Infobox parsing (limited coverage) |
| Primary labels only (no dual Entity+Person) | Cleaner for Text2Cypher generation, less confusing schema | Keep dual labels (rejected: confusing for Cypher gen) |
| Drop MENTIONS_X edges | Redundant when entities have typed labels; MENTIONS + label filter is sufficient | Keep typed edges (rejected: bloats schema, no benefit) |
| Native vector index | Required for `db.index.vector.queryNodes()` in text_search tool | Property-only storage (rejected: can't do similarity search) |
| Local embeddings (sentence-transformers) | No API key needed, works offline | Gemini embeddings (requires key, rate limits) |

## Important Notes

- Dataset is at `data/viet-wikipedia/` (1.29M articles, ~1.6GB Arrow files) — NOT in git
- Run enrichment with `uv run python -m scripts.enrich_wikidata` (needs `-m` flag for module resolution)
- Run migration with `uv run python -m scripts.migrate_schema`
- Neo4j runs as system service (not Docker) on this machine
- Embedding backend must be set to `local` (no Gemini key available): `EMBEDDING_BACKEND=local`
- NER backend should be `underthesea` for Vietnamese: `NER_BACKEND=underthesea`
