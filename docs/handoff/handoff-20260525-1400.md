## Context Summary

Building an ingestion pipeline to load 184 Vietnamese Wikipedia articles (from UIT-ViQuAD2.0 dataset) into Neo4j as a knowledge graph for multi-hop QA evaluation. The key insight: instead of using NER models (which truncate at 512 tokens and miss entities), extract entities from `[[wiki links]]` in the raw Wikipedia markup — these are human-annotated, full-coverage, high-precision entity mentions.

## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Use wiki links for entity extraction (Option B) | Full article coverage, 100% precision, no model inference, gives LINKS_TO edges for free | NER-only (truncation issue), Hybrid NER+links (marginal gain for complexity) |
| Create entities for ALL wiki links (not just our 184) | Multi-hop QA needs intermediate entities for graph traversal | Only entities linking to our 184 pages (too sparse) |
| CPU-only torch for NER pipeline | Project venv had broken CUDA deps; NER uses device=-1 anyway | Fix CUDA (complex, fragile) |
| Structure-aware chunking (v2) | Respects section boundaries, better for retrieval | Fixed-size character chunking (splits mid-sentence) |
