Problem Statement

Research Gap in Vietnamese QA

Single-hop Limitation

Existing Vietnamese QA (ViQuAD, PhoBERT) handles only extractive single-hop. No system supports multi-hop graph reasoning natively.

Hallucination Risk

57% of LLM citations are unfaithful (Liu et al., 2024). Without passage-level grounding, answers lack verifiability.

Sovereignty Gap

Current solutions depend on GPT-4/Claude APIs. No local Vietnamese KGQA exists for on-premise deployment.

Key refs: ViMQA (gated), HisGraphRAG (PACLIC 2025), Citation Faithfulness (arXiv:2412.18004)

Retrieval

Hybrid Retrieval with Cross-Encoder Reranking

1

Schema Linking — BM25/TF-IDF prunes KG schema to relevant labels/relations

2

Cypher Generation — SLM produces read-only Cypher from query + pruned schema

3

Deterministic Validation — Syntax check, write-keyword blocklist, output alias verification

4

Fallback — On invalid Cypher → hybrid fulltext (BM25 + dense vector) retrieval

5

Reranking — BAAI/bge-reranker-v2-m3 cross-encoder scores top-k (+15.5% accuracy expected)

Inspired by CyVerACT (deterministic validation) and HybridRAG (graph+text outperforms graph-only)

Architecture
Agent

Tool-Constrained ReAct Agent

4 Deterministic Tools

kg_schema() — cached JSON schema (labels, rels, props)
kg_query(cypher) — execute & return rows or errors
text_search(q, k) — hybrid BM25 + dense retrieval
get_passage(id) — raw paragraph for verification

Design Principles

  • Capped at 6 iterations — prevents runaway loops (aligned with ReflectiveRAG findings)
  • Sufficiency gating — if KG evidence empty/inconsistent → route to text_search or abstain
  • Citation tracking — every observation linked to passage_id for groundedness
  • Error refinement — compiler errors fed back as observations within the loop

Refs: ReflectiveRAG (sufficiency checks), C2RAG (adaptive re-retrieval), Agentic RAG SoK (tool-use risks)

ViWiki-MHR Generation Pipeline

1

KG Walk Extraction — 2-hop and 3-hop paths through typed graph + broken-link walks for unanswerables (BRINK-inspired)

2

Template-based QA — Vietnamese question templates per entity type × hop count × reasoning type

3

LLM Rewrite — Local model rewrites template questions for naturalness (optional stage)

4

3-Stage QC — Well-formedness filter → Grounding match (passage verification) → Deduplication

Output schema: question, answer, num_hops, reasoning_type, answerable, gold_passage_ids, cypher_query, decomposition_annotations
Quality target: Human spot-check on 1-2K samples for final validation split

Why ViQuAD 2.0?

  • Objectivity — external benchmark, not self-generated
  • Comparability — published baselines (CafeBERT: 77.5% EM, PhoBERT: 73.1% EM)
  • Abstain testing — 12K impossible questions test system's ability to say "I don't know"
  • Generative gap — measures Token F1 for generative vs extractive mismatch
